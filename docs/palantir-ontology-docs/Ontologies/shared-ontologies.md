# Shared ontologies
Shared Ontology 是一种特殊类型的 Ontology，它允许多个 [organization](/docs/foundry/security/orgs-and-spaces/#organizations) 协作处理共享 Workflow。Shared Ontology 位于专门为该 Workflow 创建的专用 shared [space](/docs/foundry/security/orgs-and-spaces/#spaces) 下。Shared Ontology 允许在 Workflow 中使用来自多个 organization 的资源，而普通 Ontology 是私有的，仅包含来自单个 organization 的资源。

A shared ontology is a special kind of ontology that allows multiple [organizations](/docs/foundry/security/orgs-and-spaces/#organizations) to collaborate on shared workflows. Shared ontologies live under a dedicated shared [space](/docs/foundry/security/orgs-and-spaces/#spaces) created specifically for the workflow. A shared ontology enables resources from multiple organizations to be used in a workflow, whereas an ontology is private to one organization and contains resources from that organization alone.
创建新的 shared space 时，会自动创建一个 Shared Ontology。它具有与 shared space 相同的 organization [markings](/docs/foundry/security/markings/) 和 [role grants](/docs/foundry/security/projects-and-roles/#roles)。

A shared ontology is automatically created when a new shared space is created. It holds the same organizations [markings](/docs/foundry/security/markings/) and [role grants](/docs/foundry/security/projects-and-roles/#roles) as the shared space.
有关如何设置安全协作 Workflow 的更多详细信息，请参阅 [跨 organization 协作](/docs/foundry/security/cross-organization-collaboration/)。

See [cross-organization collaboration](/docs/foundry/security/cross-organization-collaboration/) for more details on how to set up secure collaborative workflows.
