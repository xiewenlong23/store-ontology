# Security Rules

## 数据处理
- 不在代码中硬编码敏感信息（密钥、密码）
- 使用环境变量或配置文件管理 secrets
- JSON 数据文件不包含真实敏感数据

## API 安全
- 所有用户输入必须验证
- 使用 Pydantic 模型进行输入校验
- 错误信息不暴露内部实现细节

## 文件操作
- 使用 `Path` 模块而非字符串拼接路径
- 避免路径遍历漏洞（`../../etc/passwd`）
- 数据文件操作前验证路径在允许范围内

## CORS
- 生产环境配置明确的 CORS 白名单
- 开发环境使用代理避免 CORS 问题

## 依赖
- 定期更新依赖版本
- 使用 `pip audit` 或 `npm audit` 检查漏洞
