# AI门店大脑 - 临期商品管理

## 1. 项目概述

### 1.1 项目背景

零售门店面临临期商品管理难题：商品过期造成损耗、临期折扣策略执行不到位、人工管理效率低。通过AI助手赋能门店，实现临期商品的智能查询、自动出清任务创建与执行，降低损耗、提升运营效率。

### 1.2 项目目标

搭建基于CopilotKit + Deep Agents的AI门店大脑，为店员、店长、总部管理者提供临期商品管理的智能助手。

### 1.3 项目范围

**本期实现：**
- AI助手对话能力（查询临期商品、创建/完成出清任务）
- 临期商品本体建模与数据存储
- 多店数据隔离
- 基于 DeepSeek-chat 模型的 AI 对话
- CopilotKit v1.57.4 五大功能：Chat UI / Generative UI / Shared State / Backend Tool Rendering / Human-in-the-Loop
- Deep Agents agent 工具调用 + SummarizationMiddleware 自动压缩上下文
- **Context 自动压缩：SummarizationMiddleware 解决多轮对话 BadRequestError**
- **DeltaChannel：checkpoint 增长从 O(N²) 降至 O(N)**

**后续扩展：**
- 外部系统对接（ERP、POS）
- 移动端支持
- 数据库持久化
- CopilotKit v2 正式版升级

### 2.4 本体驱动架构（v0.3.0）

```
ontology/store.ttl          ← 本体语义定义（标准 TTL 格式）
       ↓                            Object Types / Link Types / Action Types
ontology/parser.py          ← 解析 TTL → EntityRegistry
       ↓                            7 对象类型 + 6 关系类型 + 2 动作类型
ontology/tools.py           ← 3 个通用工具：
       ├── query_entity()   ← 查任意实体（读注册表知查什么文件）
       ├── create_entity()  ← 创建任意实体
       └── traverse_relation() ← 遍历关系链（如 Store→Employee via store_id）
       ↓
main.py                     ← 挂载通用工具 + 专用工具（get_near_expiry_products 等）
       ↓                           动态生成系统提示 (build_ontology_prompt)
create_deep_agent()          ← Deep Agents 自动处理工具调用循环
       ↓                           SummarizationMiddleware 自动压缩上下文
LangGraphAgent               ← ag_ui_langgraph，暴露 AG-UI SSE 端点
```

**核心原则：** 新实体类型只需在 `store.ttl` 中加一行 Turtle 定义，无需写 Python 代码。AI 通过解析后的系统提示自动理解实体语义。实体实例数据按现状存储为 JSON 文件。`query_entity` 根据实体注册表自动路由到对应 JSON 文件。

## 3. 本体建模

### 3.1 本体论概述

本体论(Ontology)是对现实世界概念的形式化表示。参考 Palantir Foundry Ontology 标准，采用 TTL（Turtle）格式存储语义定义，JSON 文件存储实例数据。

**语义层次：**
| 层次 | 格式 | 内容 |
|------|------|------|
| TBox（术语层） | `ontology/store.ttl` | Object Types / Link Types / Action Types 的定义 |
| ABox（实例层） | `data/*.json` | 具体的门店、员工、商品等实例数据 |
| AI 理解层 | `build_ontology_prompt()` | 从 TTL 自动生成系统提示 |

| 类型 | 说明 |
|------|------|
| **Object Types** | 对象类型，实体定义（如 Store, Product, Employee） |
| **Link Types** | 链接类型，对象之间的关系 |
| **Action Types** | 动作类型，对对象的操作集合（含业务逻辑） |
| **Functions** | 函数，业务逻辑实现（TypeScript/Python） |
| **Interfaces** | 接口，对象类型的抽象形状 |

**核心概念：**
- **Object Type Definition**：类型级别的定义（如"所有员工"）
- **Object Instance**：具体的实例（如"张三"）
- **Action Type**：操作的定义（如"出清"）
- **Action Instance (= Task)**：Action Type 的具体执行实例（如"出清任务 #001"）

**Parameters（参数）：**

**Submission Criteria（提交条件）：**

| 条件类型 | 表达式 | 失败消息 |
|----------|--------|----------|
| 参数校验 | quantity > 0 | 出清数量必须大于0 |
| 参数校验 | quantity <= stock_quantity | 出清数量不能超过库存 |
| 参数校验 | assignee_id exists | 负责人不存在 |
| 状态校验 | product.status != 'expired' | 已过期商品不能出清 |

**Side Effects（副作用）：**

| 类型 | 配置 | 说明 |
|------|------|------|
| Notification | 通知店长 | 新建出清任务时通知 |
| Webhook | 记录日志 | 任务状态变更时记录 |

### 5.1 AI对话接口

**前端API Route:** `POST /api/copilotkit` (Next.js)

由 `CopilotRuntime` + `LangGraphHttpAgent` 代理到 Deep Agent 服务。

```typescript
// app/api/copilotkit/route.ts
import { CopilotRuntime, ExperimentalEmptyAdapter, copilotRuntimeNextJSAppRouterEndpoint } from "@copilotkit/runtime";
import { LangGraphHttpAgent } from "@copilotkit/runtime/langgraph";
import { NextRequest } from "next/server";

const serviceAdapter = new ExperimentalEmptyAdapter();
const runtime = new CopilotRuntime({
    agents: {
        default: new LangGraphHttpAgent({
            url: process.env.LANGGRAPH_DEPLOYMENT_URL || "http://localhost:8123/api/copilotkit",
        }),
    }
});

export const POST = async (req: NextRequest) => {
    const { handleRequest } = copilotRuntimeNextJSAppRouterEndpoint({
        runtime,
        serviceAdapter,
        endpoint: "/api/copilotkit",
    });
    return handleRequest(req);
};
```

## 6. 功能模块

### 6.1 AI助手模块

#### 6.1.1 功能描述

通过自然语言对话，为用户提供临期商品管理服务。Agent 可调用后端工具查询数据、创建出清任务，并在关键操作前请求用户确认。

#### 6.1.2 CopilotKit 五大功能实现（实际）

##### Generative UI（生成式UI）

**实现方式：** `<CopilotKit renderToolCalls={[...]}>` prop。

```tsx
// app/layout.tsx - 在 CopilotKit Provider 上注册渲染器
'use client'
import { CopilotKit } from '@copilotkit/react-core'

function ProductCard({ text }: { text: string }) {
  return (
    <div style={{padding:12,background:'#f0fdf4',borderRadius:8}}>
      <strong>📋 Generative UI</strong>
      <pre>{text}</pre>
    </div>
  )
}

export default function RootLayout({ children }) {
  const renderToolCalls = useMemo(() => [
    {
      name: 'get_near_expiry_products',
      render: ({ status, result }) => {
        if (status === 'executing') return <div>🔍 查询中...</div>
        if (status === 'complete' && result) return <ProductCard text={result} />
        return <></>
      },
    },
    {
      name: 'create_clearance_task',
      render: ({ status, result }) => {
        if (status === 'executing') return <div>⚙️ 创建中...</div>
        if (status === 'complete' && result) return <TaskCard text={result} />
        return <></>
      },
    },
  ], [])

  return (
    <html lang="zh-CN">
      <body>
        <CopilotKit runtimeUrl="/api/copilotkit" agent="default" renderToolCalls={renderToolCalls}>
          {children}
          <CopilotPopup />
        </CopilotKit>
      </body>
    </html>
  )
}
```

**注意：** `useCopilotAction` hook 内部使用 v2 context，在 v1 provider 下不兼容。正确方式是通过 `<CopilotKit renderToolCalls={[...]}>` prop 注入渲染器数组。

##### Shared State（共享状态）

**实现方式：** `useCoAgent({ name, initialState })`。


```python
def chat_node(state: AgentState) -> AgentState:
    system_prompt = f"""当前用户选择的门店ID是: {state.get('selected_store', 'store_001')}
用户手动切换门店时，该值会自动更新。"""
```

##### Human-in-the-Loop（人机协同）

**实现方式：** LangGraph 原生 `interrupt()`，无需 CopilotKit hook。

```python
# backend/main.py - Graph 节点
from langgraph.types import interrupt

def human_approval_node(state: AgentState) -> AgentState:
    """人工审批节点 - 暂停等待用户确认"""
    preview = state["pending_action"]
    # 中断执行
    approval = interrupt({
        "type": "confirm_clearance",
        "message": f"即将为门店 {preview['store_name']} 创建出清任务，请确认",
        "preview": preview,
    })
    if approval.get("approved"):
        return state  # → execute_action
    else:
        return {**state, "messages": [AIMessage(content="用户取消了任务")]}

def execute_action_node(state: AgentState) -> AgentState:
    """执行审批通过的操作"""
    pending = state.get("pending_action")
    # 实际创建出清任务
    ontology.create_clearance_task(task)
    return {**state, "messages": [AIMessage(content="✅ 出清任务已创建！")]}
```

**Graph 路由：**
```
chat → tools → should_approve → human_approval (⏸️ interrupt)
                                      ↓ 用户确认
                                execute_action → chat
```

### 6.1.3 LangGraph Agent 架构

#### Graph 结构

```
                    ┌─────────┐
         ┌──────────│  chat   │◄──────────────────┐
         │          └────┬────┘                    │
         │       tool    │                         │
         │    ┌──────────▼──────────┐              │
         │    │       tools         │              │
         │    └──────────┬──────────┘              │
         │               │ should_approve          │
         │     ┌─────────▼─────────┐               │
         │     │  human_approval   │               │
         │     │   ⏸️ interrupt()  │               │
         │     └─────────┬─────────┘               │
         │      approve  │  reject                 │
         │     ┌─────────▼─────────┐               │
         │     │  execute_action   │               │
         │     └─────────┬─────────┘               │
         │               │                         │
         └───────────────┴─────────────────────────┘
```

#### 节点说明

| 节点 | 功能 | 输入 | 输出 |
|------|------|------|------|
| `chat` | LLM + 工具调用 | `messages`, `selected_store` | AIMessage (含 tool_calls) |
| `tools` | 执行工具函数 | tool_calls | ToolMessage (结果) |
| `human_approval` | 中断等待用户确认 | `pending_action` | `interrupt()` 事件 |
| `execute_action` | 执行审批通过的操作 | `pending_action` | 实际创建任务 |
| `should_continue` | 路由：是否需要工具 | 最后消息 | `"tools"` / `"end"` |
| `should_approve` | 路由：是否需要审批 | 工具结果 | `"approve"` / `"end"` |

#### AgentState

```python
class AgentState(TypedDict):
    messages: Annotated[List[Any], "messages"]     # 对话历史
    selected_store: str                             # Shared State
    pending_action: Optional[Dict[str, Any]]        # HITL 待审批操作
```

### 6.1.4 对话能力

| 能力 | 示例对话 | 触发工具 |
|------|----------|----------|
| 查询任意实体 | "store_001 有哪些员工" | `query_entity(type="employee", store_id="store_001")` |
| 遍历关系 | "张三属于哪个门店" | `traverse_relation(Employee, emp_001, manages)` |
| 查询临期商品 | "查看门店的临期商品" | `get_near_expiry_products(store_001)` |
| 创建任务 | "帮我创建一个出清任务" | `create_clearance_task(store_001, nep_001, 50)` → ⏸️ HITL |
| 门店摘要 | "给我门店概览" | `get_store_summary(store_001)` |
| 折扣查询 | "折扣规则是什么" | `query_entity(type="discount_rule")` |

**工具设计原则：**
- **通用工具（3个）**：`query_entity`、`traverse_relation`、`create_entity` — 基于 TTL 本体注册表，覆盖所有实体类型
- **专用工具（2个）**：`get_near_expiry_products`、`create_clearance_task` — 包含业务逻辑的格式化输出
- **聚合工具（1个）**：`get_store_summary` — 一次调用返回门店全景视图
- 新实体类型只需在 `ontology/store.ttl` 加一行，通用工具自动覆盖

```
     ┌──────────┐
```
