## 关键占位符填法

### `locator_field`（Action，新机制，最易填错）

决定 executor 如何定位 target 记录。规则：
- **target 是工作流对象**（Task / RepairTicket / ...）→ 填该对象的 id 参数名：`task_id` / `ticket_id`。
- **target 是标的物**（NearExpiryProduct / Equipment / ...）→ 填 `target_id`（或不填，默认）。
- **一个 Action 同时操作工作流对象 + 标的物**（如 deduct_stock）→ target_object_type 是谁，locator_field 就对应谁。deduct_stock 的 target 是 NearExpiryProduct → `locator_field: target_id`（工作流对象 Task 的状态在 submission_criteria 里用 `task.status` 单独查）。

判断口诀：**locator_field 跟 target_object_type 走，不跟"你想改的东西"走。**

### `workflow_object_type` + `workflow_object_id_field`（config.py）

- `workflow_object_type`：工作流主对象类名（如 `RepairTicket`）。无工作流填 None。
- `workflow_object_id_field`：定位工作流对象的参数名（如 `ticket_id`）。submission_criteria 里写 `<workflow_object_lower>.status` 时，executor 用这个字段去 registry 查对象。

### `prefix`（TTL）

从 `@prefix <prefix>: <iri> .` 行读取（parser 动态解析）。建议用 vertical 名（如 `repair:`）。clearance 历史用 `store:`。

## 每个模板的常见错误

| 模板 | 常见错误 | 正确做法 |
|---|---|---|
| ontology.ttl | TTL 块不以 ` .` 行结尾 → parser 漏解析 | 每个块最后一行 ` .` |
| ontology.ttl | via 字段不属于 domain 也不属于 range | via 必须在两者之一的 properties 里（§4.3） |
| ontology.ttl | 枚举写成裸字符串 | 用枚举类型名：`status:TicketStatus` |
| action.yaml | locator_field 跟"想改的对象"走 | 跟 target_object_type 走 |
| action.yaml | transforms 用字符串表达式 | 用结构化 `{op, field, by}`，禁止 eval |
| action.yaml | submission 条件 field 写 `task.status` 但 vertical 工作流对象不是 Task | 写 `<workflow_object_lower>.status`（如 `repair_ticket.status`） |
| state_machine.py | 终态漏列 terminal_states | 终态必须列入，否则能再迁移 |
| seed json | 外键指向不存在的实例 | 跑 FK 校验脚本 |
| config.py | data_dir 指错（指到 clearance 的 data/） | 指向 `data/<vertical_name>/` |
| config.py | tools_module 指了不存在的模块 | 无专属工具时设 None |
| skill md | allowed_tools 列了不存在的工具 | 只列真实存在的（内核 + 本 vertical） |

## 占位符替换技巧

批量替换时，建议按这个顺序（依赖链：prefix → 类名 → 字段 → 参数）：
1. `{{prefix}}` / `{{namespace_iri}}`（全局替换，先定）
2. `{{ClassName}}` / `{{vertical_name}}`（每个类/vertical 一次）
3. `{{field}}` / `{{param}}` / `{{locator_field}}`（逐个 Action）
4. 条件块 `{{#...}}...{{/}}`：满足保留，不满足整段删。

完整填好的例子见 `04-worked-example-设备维修工单.md`。
